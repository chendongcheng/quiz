# 线上关怀用户问题库 V6

使用原则：
- 单次对话优先选 3—5 个问题，每次只提出一个主问题，根据回答再自然追问。
- 首次联系从称呼、联系时间、兴趣、饮食偏好等低压力话题开始，不连续询问健康、家庭、财务等敏感内容。
- `必要时`问题只在用户主动提到相关情况、前序回答触发，或当前服务确实需要时使用，不作为固定问卷逐项询问。
- 提问保持中立，不预设用户“做得不好”、不以家人意见代替本人感受，也不使用劝说、评判或诊断式表达。
- 涉及胸痛、明显呼吸困难、意识异常、自伤、受伤害或失联风险时，停止普通关怀流程，按对应安全路径响应。
- 不询问或记录密码、验证码、银行卡完整号码、身份证完整号码等可直接用于账户操作的信息。

建议对话路径：
1. 建立熟悉感：称呼方式 → 联系时间 → 聊天偏好。
2. 了解日常：作息、饮食、兴趣、活动，选择其中 1—2 个自然展开。
3. 识别需要：从用户刚提到的困难进入出行、提醒、陪伴或生活服务。
4. 必要时关怀：只围绕已出现的健康、安全或情绪线索继续了解。
5. 形成行动：确认一个具体、可执行的提醒、联系或支持安排。

标记说明：
- `初次了解`：适合首次填写或低压力场景，问题轻松、容易回答。
- `熟悉后`：适合在基础信息之后继续填写，能了解更具体的生活状态。
- `必要时`：由相关回答、服务场景或安全线索触发，不在常规聊天中批量展示。
- `低隐私`：日常偏好、生活习惯、兴趣爱好等。
- `中隐私`：家庭联系、情绪状态、生活状态等。
- `高隐私`：健康、用药、体检、紧急联系人、敏感家庭关系等。
- `重点关注`：回答可能需要进一步确认或安排后续支持。
